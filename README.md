# Лабораторная работа 25 (скриншоты в 25.docx)
## Было проведено тестирование приложения из лабораторной 21, в результате чего:
- Настроены зависимости для тестов: junit:4.13.2, kotlinx-coroutines-test в build.gradle.
- Написаны Unit-тесты для UseCase (domain): GetNotesUseCase (проверка получения списка заметок, пустого списка) и AddNoteUseCase (проверка валидации: при пустом заголовке возвращается ошибка). Репозиторий заменён на фейковый (in-memory коллекция).
- Написаны Unit-тесты для репозитория (data): создан FakeNoteDao, который хранит данные в mutableListOf. Проверены сценарии добавления и удаления заметок через NotesRepositoryImpl.
- Написан интеграционный тест с реальной Room: использован Room.inMemoryDatabaseBuilder, проверена совместная работа NotesRepositoryImpl с настоящим NoteDao (добавление заметки → чтение списка → заметка присутствует).
- Написаны Unit-тесты для ViewModel (NotesViewModel): использован StandardTestDispatcher, проверены состояния загрузки (isLoading), успешная загрузка списка, обработка ошибки (через фейковый UseCase с Result.failure).
- Все тесты изолированы от внешних зависимостей (моки/фейки), корутины управляются через runTest и advanceUntilIdle().
- В отчёте отмечены различия между Unit- и интеграционными тестами, а также как тестирование помогло убедиться в корректности архитектуры.
